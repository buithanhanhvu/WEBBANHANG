dự án này tôi tính thêm phần test để cho vào cv xin việc của tôi trong tester nhưng nó đang thiếu 1 số thứ Unit Test (kiểm thử mức đơn vị) là việc lập trình viên tự viết code để kiểm tra độc lập từng hàm (function) hoặc phương thức (method) nhỏ nhất trong dự án, đảm bảo logic xử lý luôn chính xác trước khi ghép nối toàn bộ hệ thống. Với các dự án Backend sử dụng Java Spring Boot hay Node.js, bạn sẽ dùng các thư viện như JUnit, Mockito hoặc Jest để test các Controller, Service và API mà không cần chạy cả server thực tế. Để tiết kiệm thời gian, các lập trình viên hiện nay thường tận dụng tối đa các trợ lý AI như Cursor hoặc GitHub Copilot để tự động sinh (generate) ra các đoạn code Unit Test này một cách nhanh chóng. Việc thể hiện kỹ năng Unit Test trong CV sẽ chứng minh bạn có tư duy viết code cẩn thận, chú trọng chất lượng mã nguồn và biết cách ngăn chặn "bug" ngay từ lúc đang code thay vì đợi đến khi đưa cho team Tester.Thực chất, trong cùng source code đó, bạn chỉ cần tạo một thư mục riêng (thường là test nằm song song với mã nguồn chính). Tại đây, bạn viết các đoạn code ngắn để gọi và chạy thử trực tiếp các hàm hoặc API mà bạn vừa code xong. Đoạn code test này sẽ tự động bơm dữ liệu giả vào hàm, sau đó kiểm tra xem kết quả trả về có chính xác như logic bạn mong muốn hay không, giúp phát hiện lỗi sai ngay lập tức trên từng dòng code mà không cần phải chạy toàn bộ hệ thống lên để test thủ công.
Tester (QA/QC): Các dự án kiểm thử ấn tượng thường mô phỏng lại quá trình đánh giá chất lượng cho một nền tảng thương mại điện tử hoặc hệ thống bán lẻ quy mô lớn. Điểm sáng trong CV sẽ là sự kết hợp giữa kiểm thử thủ công và tự động, sử dụng Postman để kiểm tra tính toàn vẹn của API, Selenium cho các kịch bản UI, và quản lý lỗi hệ thống qua MantisBT. Việc chứng minh khả năng áp dụng linh hoạt kiểm thử hộp đen, hộp trắng, hộp xám, cùng kỹ năng viết test case rõ ràng, logic sẽ đánh giá chính xác năng lực thực chiến của ứng viên.
